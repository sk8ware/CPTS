## Resumen

Este módulo presenta la ruta de aprendizaje **Penetration Tester de Hack The Box Academy**, diseñada para desarrollar las habilidades necesarias para realizar evaluaciones de seguridad profesionales.

La ruta cubre pruebas de penetración:

- Externas.
    
- Internas.
    
- Sobre Active Directory.
    
- En aplicaciones web.
    

El aprendizaje sigue una metodología práctica basada en **“aprender haciendo”**, comprendiendo no solamente cómo ejecutar una técnica, sino también:

- Por qué existe una vulnerabilidad.
    
- Cómo identificarla.
    
- Cómo explotarla.
    
- Cómo detectarla.
    
- Cómo mitigarla.
    
- Cómo documentarla correctamente.
    

## Metodología de aprendizaje

HTB Academy recomienda desarrollar una metodología propia, organizada y repetible que pueda aplicarse a diferentes tipos de pentest.

La ruta recorre las principales fases de una evaluación:

1. Reconocimiento.
    
2. Enumeración.
    
3. Evaluación de vulnerabilidades.
    
4. Explotación.
    
5. Movimiento lateral.
    
6. Post-explotación.
    
7. Escalada de privilegios.
    
8. Documentación y reporte.
    

## Consideraciones legales y éticas

Las pruebas de penetración deben realizarse únicamente sobre sistemas autorizados y dentro de un alcance definido por escrito.

Antes de comenzar una evaluación se debe contar con:

- Contrato o autorización firmada.
    
- Alcance del pentest.
    
- Direcciones IP, dominios o aplicaciones autorizadas.
    
- Reglas de enfrentamiento.
    
- Horarios permitidos para las pruebas.
    

Nunca se deben escanear, explotar o interactuar con sistemas externos sin autorización. También se debe evitar ejecutar técnicas que puedan provocar interrupciones o daños en los servicios.

## Principio fundamental

> Un pentester debe trabajar de forma ética, mantenerse dentro del alcance autorizado, evitar causar daños y documentar todas sus actividades.
